# 書籍開發路線

本書跟隨配套系統能力逐步成熟，但不照程式 Phase 原樣排列教學順序。新手必須
先理解市場、資料與會計，再接觸完整策略研究與上線操作。

## R0：書籍地基

- [x] 定義讀者、教學目標與風險邊界
- [x] 建立全書目錄、寫作規範與系統對照表
- [x] 建立章節模板
- [x] 完成序章與第一個知識章
- [ ] 決定出版工具鏈與輸出格式

## R1：可由 v0.3.0 驗證的基礎教材

- [ ] 完成起步篇
- [ ] 完成市場篇
- [ ] 完成資料篇，實際驗證 `quant-data`
- [ ] 完成交易核心篇，以 Phase 3 測試與 golden 帳本作案例
- [ ] 建立術語表與版本相容表

## R2：Phase 4／4.5 引擎與正式入口

- [ ] Phase 4 發布後驗證引擎、runner、多腿與風控章節
- [ ] Phase 4.5 發布後，以正式 Foundation 入口建立第一條完整教學實驗
- [ ] 將預覽章節轉為可操作，保存實際輸出與 provenance

## R3：研究與 Backtest Server

- [ ] Phase 5 發布後完成回測、穩健性、報告與 MCP 章節
- [ ] 建立一個中性策略與一個網格策略的教學案例
- [ ] 驗證所有範例均使用正式 API／CLI，沒有書籍專用交易邏輯

## R4：Paper、Live 與出版驗收

- [ ] Phase 7 發布後完成 Paper 操作與恢復演練
- [ ] Phase 8 發布後完成 Live、對帳、確認與緊急停止章節
- [ ] 重新查證交易所規則、API、費率、安全與法規相關內容
- [ ] 由新手試讀並記錄卡點，完成技術與文字校對
- [ ] 對指定配套 tag 全量重跑書中命令後再發布
