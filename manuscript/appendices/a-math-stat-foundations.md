## 先固定對象、時點與單位

一個數字至少要和三件事一起讀：它描述什麼、在哪個時點成立、使用什麼單位。
`20,000` 若沒有標示，可能是價格、名義價值、時間戳或列數；它本身不是完整輸入。

| 寫法 | 本書中的讀法 | 使用前提 |
|---|---|---|
| `x` | 一個已定義的量 | 同一段推導中不偷換意義 |
| `x_t` | 量 `x` 在時點或序號 `t` 的值 | `t` 的時區、頻率與事件語義已固定 |
| `x_i` | 集合或序列中第 `i` 個值 | 索引範圍與排序規則已固定 |
| `Δx` | 明示區間內的變化量 | 說清楚是 `after - before` 或其他方向 |
| `x_min`、`x_max` | 允許範圍的下界與上界 | 邊界是否包含必須另寫 |
| `\sum_i x_i` | 對指定索引集合逐項相加 | 每一項可相加且沒有重複入帳 |

等號 `=` 表示兩側在既定前提下相等；`:=` 表示「在此定義為」。箭頭 `→` 常用來
表示狀態轉移，不等於數學相等。第 17 章的訂單狀態若由 `NEW → FILLED`，不能讀成
兩個狀態同時成立。

## 單位是第一個錯誤偵測器

乘除時先保留單位，再消去共同項。價格乘數量得到名義價值：

\\[
20{,}000\ \frac{\mathrm{USDT}}{\mathrm{BTC}}
\times 0.25\ \mathrm{BTC}
=5{,}000\ \mathrm{USDT}
\\]

加減只對相同單位有意義。`5,000 USDT + 0.25 BTC` 不能直接相加；必須先選一個
帶時點與來源的 BTC 重估價，把 BTC 換算成 USDT。這個換算會加入估值假設，不能
再說結果只是原始資產流。

比率的分子、分母若單位相同，結果通常無單位，例如費率與參與率。無單位不代表
沒有口徑：

\\[
T=\frac{N\_{\mathrm{entry}}+N\_{\mathrm{exit}}}
        {E\_{\mathrm{initial}}}
\\]

換手率 `T` 的分母若改成平均權益，結果就變成另一個指標。名稱相同不能補回遺失
的分母定義。

## 正負號、絕對值與方向

帶方向數量以正負號保存方向，絕對值只保留大小：

\\[
Q>0\Rightarrow\text{多頭},\qquad
Q<0\Rightarrow\text{空頭},\qquad
|Q|\ge 0
\\]

因此線性永續的名義價值 `N=|Q|m` 一律非負，價格損益
`U=(m-p_entry)Q` 則保留 `Q` 的方向。不能因為報告只想顯示正數，就把損益或現金流
也取絕對值。

絕對值也用於「距離」或虧損大小，但必須保留原始方向。例如附錄 C 的賠率分母
可使用 `|L|`，同一筆 `L` 在帳本仍是負數。

## 不等式、區間與未定義

`<`、`>` 不包含邊界；`<=`、`>=` 包含邊界。半開區間
`[0, 50,000)` 表示 `0 <= x < 50,000`，所以 `50,000` 不屬於這一檔。

分母為零時，先停止而不是硬填 `0`。例如：

- `MM / MB` 在 `MB <= 0` 時不應顯示成正常保證金率；
- 相對變化 `(x_1-x_0)/x_0` 在 `x_0=0` 時未定義；
- 加權平均在權重總和為零時未定義；
- 部分成交配置在 `q_remaining,before=0` 時不應再執行。

「未定義」是可操作的 fail-closed 結果，不是缺值可以隨意補零的理由。

## 比例、百分比與基點

比例在計算中用十進位值：

\\[
1\%=0.01,\qquad
1\ \mathrm{bp}=0.0001=0.01\%
\\]

因此 `5 bps = 0.0005 = 0.05%`。程式中若費率欄位要求十進位比例，輸入 `5`
會放大一萬倍；輸入 `0.05` 則是 `5%`，不是 `0.05%`。

百分點與相對百分比不同。勝率由 `50%` 到 `55%` 是增加 `5` 個百分點；相對原值
則增加 `(0.55-0.50)/0.50 = 10%`。報告必須說明採哪一種。

基點只適合表達率的差，不是價格或 PnL 單位。把 `10 bps` 費率換成現金成本時，
仍要乘上明示名義價值：

\\[
F=N\times 10\times0.0001
\\]

## 十進位精度與顯示捨入

金額、價格、數量與費率從字串建立 `Decimal`：

```python
from decimal import Decimal

price = Decimal("20000.10")
qty = Decimal("0.015")
fee_rate = Decimal("5") * Decimal("0.0001")

notional = price * qty
fee = notional * fee_rate
```

不要先寫 `0.1` float 再傳入 `Decimal`。二進位浮點近似已在第一步發生，之後改型別
不會恢復原本的十進位字串。

計算值與顯示值分開保存。先用未捨入值完成守恆、損益與風險 oracle，最後才依
明示規則產生顯示值：

```python
from decimal import Decimal, ROUND_HALF_EVEN

raw = Decimal("16064.25702811245")
display = raw.quantize(Decimal("0.01"), rounding=ROUND_HALF_EVEN)
assert display == Decimal("16064.26")
```

顯示為 `16064.26` 不代表帳本可以丟棄 `raw`。不同交易所規則還可能要求向下、向上
或朝零截斷；沒有規則證據時，不把一般四捨五入冒充合法訂單量化方式。

## 網格合法性不是「四捨五入後差不多」

價格與數量在離散網格上時，可用餘數檢查：

\\[
\begin{aligned}
(p-p\_{\min})\bmod\Delta p&=0 \\\\
(q-q\_{\min})\bmod\Delta q&=0
\end{aligned}
\\]

`Δp`、`Δq` 必須為正，且 `p_min`、`q_min` 與網格來源一致。餘數非零表示輸入不在
網格上；它不授權程式悄悄修價或改量。第 16 章還要求同時檢查上下界與最小名義
價值，通過餘數只是其中一個 gate。

## 加總、橋接式與避免重複入帳

帳本核對常使用「期末 = 期初 + 所有變化」：

\\[
X\_{\mathrm{after}}
=X\_{\mathrm{before}}+\sum\_i\Delta X\_i
\\]

使用前逐項確認：

1. 每個 `ΔX_i` 與 `X` 單位相同；
2. 正負號採同一帳戶觀點；
3. 相同事件只出現一次；
4. 起訖時點沒有重疊或缺口。

「由實際成交價算出的毛損益」已包含 spread 與滑點造成的價格差，就不能再扣一次
同一價格摩擦。相同原則也適用 funding、fee、realized PnL 與 reservation：分類欄
可以解釋總額，但不能在總額外再入帳。

## 比例配置與容量上限

按比例配置時，分子與分母必須描述同一個母量。第 19 章把一次部分成交對應的
reservation 配置為：

\\[
A\_{\mathrm{allocated}}
=R\_{\mathrm{before}}
\times\frac{q\_{\mathrm{fill}}}
             {q\_{\mathrm{remaining,before}}}
\\]

前提是 `0 < q_fill <= q_remaining,before` 且 `R_before >= 0`。配置後還要保存
`R_after = R_before - A_allocated`；單看比例正確不能證明資金帳本完整。

容量情境 `q_capacity = D_scenario × c_participation` 同樣只是明示深度與參與率下的
上限。`D_scenario` 若不是當下可成交深度，結果不能升格為市場容量事實。

## 簡單平均與加權平均

`n` 個數值的算術平均為：

\\[
\bar{x}=\frac{1}{n}\sum\_{i=1}^{n}x\_i,
\qquad n>0
\\]

每個觀測的權重相同時才使用簡單平均。成交均價通常要按實際成交數量加權：

\\[
\bar{p}\_{\mathrm{weighted}}
=\frac{\sum\_i q\_i p\_i}{\sum\_i q\_i},
\qquad q\_i\ge0,\quad\sum\_iq\_i>0
\\]

例如 `0.01 BTC @ 20,000` 與 `0.03 BTC @ 19,000` 的均價是
`19,250 USDT/BTC`，不是兩個價格的簡單平均 `19,500`。未成交掛單的 `q_i=0`，
不能先加入成交均價。

若資料是帶方向成交，先明示本輪是在增加同方向庫存、減倉或反向；不能把負權重
直接塞進這個非負權重公式。

## 計數、最小值、最大值與有限樣本

計數 `n` 先要求清楚的集合邊界。兩個 UTC 日的 `1h` bar 理論列數是 `48`，前提包括
起訖端點、時區、頻率與缺口規則；檔案剛好有 `48` 列仍不能證明沒有重複或亂序。

K 線結構檢查使用最小值與最大值：

\\[
L\le\min(O,C)\le\max(O,C)\le H
\\]

這只能證明單列 OHLC 結構相容。它不證明價格真實、資料完整、來源可信或策略有效。

樣本比例把符合條件的觀測數除以總觀測數：

\\[
\hat{p}=\frac{k}{n},
\qquad 0\le k\le n,\quad n>0
\\]

帽號提醒這是樣本摘要，不是未來機率的保證。第 1、3、6 章提到的高勝率、年化
報酬或回測結果，若缺少樣本範圍、成本、樣本外證據與失效條件，不能直接支持
實盤決定。

## 期望值不是下一筆結果

對有限 outcome `x_i` 與機率 `p_i`：

\\[
\mathrm{E}[X]=\sum\_i p\_i x\_i,
\qquad p\_i\ge0,\quad\sum\_i p\_i=1
\\]

期望值是指定分布下的加權平均，不代表下一筆一定等於該數字，也不顯示發生順序、
最大損失或強平路徑。成本若已包含在每個 `x_i`，不能再從期望值扣第二次。

W1 只需要這個有限 outcome 定義。母體推論、信賴區間、波動率估計或其他進階統計
若沒有相符正文與證據，不從本附錄外推。

## AND、OR 與集合條件

布林 gate 的結果只有 true／false，不能用平均分數取代：

\\[
R=Q\land C\land N\land F\land S\land V\land M
\\]

`R` 為 true 的必要條件是右側每一項都為 true。其餘六項都通過、只要一項失敗，結果
仍是 false。`A OR B` 則至少一項為 true；括號決定混合 AND／OR 的結合順序，不能省略後
靠直覺補回。

PIT universe 的成員資格也是帶時點的集合條件：

\\[
\mathrm{onboard\_ts}\le\mathrm{as\_of}
\land
(\mathrm{delist\_ts}=\mathrm{None}
\lor\mathrm{delist\_ts}>\mathrm{as\_of})
\\]

`as_of` 改變，集合就可能改變。使用今天的成員名單回填歷史資料，並不是同一個集合。

## 一組可重現的 Decimal 核對

以下固定數字同時核對名義價值、基點、加權平均與按比例配置；它們是數學 oracle，
不是市場資料或交易建議：

```python
from decimal import Decimal as D

notional = D("20000.10") * D("0.015")
rate = D("5") * D("0.0001")
fee = notional * rate

weighted_price = (
    D("20000") * D("0.01") + D("19000") * D("0.03")
) / D("0.04")
allocated = D("800") * D("0.01") / D("0.04")

assert notional == D("300.00150")
assert rate == D("0.0005")
assert fee == D("0.150000750")
assert weighted_price == D("19250")
assert allocated == D("200")
print("appendix-a-math-oracle=PASS")
```

若任一 assertion 失敗，先檢查字串輸入、單位、費率換算、分母與入帳方向，不要
用顯示捨入掩蓋差異。
